FILES.txt — messageEngine repository file index
================================================

BUILD / CONFIG
--------------
Makefile
  GNU make build file; compiles all libraries, server, client, and test binaries
  with the full strict warning set (-Wall -Wextra -Wpedantic -fno-exceptions -fno-rtti).
  Targets: all, server, client, tests, run_tests, lint, cppcheck, scan_build,
  static_analysis, coverage, coverage_show, clean.

docs/CMakeLists.txt
  CMake equivalent of the Makefile (provided for IDEs/CI that prefer cmake).

.gitignore
  Excludes the build/ output directory from version control.

.cppcheck-suppress
  Cppcheck suppression list; each entry has a justification comment and a
  DEFECT_LOG.md reference.

src/.clang-tidy
  Clang-Tidy configuration for production code (src/); strict rules matching
  JPL Power of 10, MISRA C++:2023, and F-Prime style subset.

tests/.clang-tidy
  Clang-Tidy configuration for test code (tests/); slightly relaxed relative
  to src/ to accommodate test-framework idioms (see CLAUDE.md §3).

check_requirements.txt
  Checklist used to verify that all numbered requirements ([REQ-x.x]) are
  implemented and tested; cross-referenced against TRACEABILITY_MATRIX.md.

SOURCE — src/core/   (message layer; no OS or socket dependencies)
-------------------------------------------------------------------
Types.hpp
  All shared enumerations (MessageType, ReliabilityClass, Result, Severity,
  TransportKind, OrderingMode), compile-time capacity constants
  (MSG_MAX_PAYLOAD_BYTES=4096, MSG_RING_CAPACITY=64, ACK_TRACKER_CAPACITY=32,
  DEDUP_WINDOW_SIZE=128, MAX_RETRY_COUNT=5, IMPAIR_DELAY_BUF_SIZE=32, etc.),
  and the NodeId typedef.

Assert.hpp
  NEVER_COMPILED_OUT_ASSERT macro: always compiled in (never NDEBUG-disabled);
  debug/test builds call abort(); production logs FATAL and triggers a
  controlled component reset.

Logger.hpp
  Header-only, heap-free logger; writes severity-tagged lines to stderr using
  a fixed 512-byte stack buffer. No dynamic allocation.

MessageEnvelope.hpp
  The standard wire envelope struct (message_type, message_id, timestamp_us,
  expiry_time_us, source_id, destination_id, priority, reliability_class,
  payload[4096]) plus inline helpers: envelope_init, envelope_copy,
  envelope_valid, envelope_is_data, envelope_is_control, envelope_make_ack.

ChannelConfig.hpp
  ChannelConfig and TransportConfig POD structs with per-channel priority,
  reliability mode, queue depth, timeouts, and impairment toggle.
  Provides channel_config_default() and transport_config_default() inline
  helpers.

TransportInterface.hpp
  Pure-virtual abstract base class (init / send_message / receive_message /
  close / is_open) that decouples the message layer from all socket backends.

RingBuffer.hpp
  Header-only, statically allocated, lock-free SPSC ring buffer for
  MessageEnvelope (capacity MSG_RING_CAPACITY=64); push / pop / peek with
  ERR_FULL / ERR_EMPTY result codes; uses std::atomic<uint32_t> with
  acquire/release ordering.

Serializer.hpp / Serializer.cpp
  Deterministic, big-endian (network byte order) wire serialization of
  MessageEnvelope into a fixed 44-byte header + variable payload; uses manual
  bit-shift I/O (no htons/ntohl). Validates padding fields and payload bounds
  on deserialization.

MessageId.hpp / MessageId.cpp
  Monotonically incrementing 64-bit message-ID generator (MessageIdGen) with
  optional seed support; never returns 0 (reserved as invalid).

Timestamp.hpp / Timestamp.cpp
  Helpers using CLOCK_MONOTONIC: timestamp_now_us() returns current time in
  microseconds; timestamp_expired() checks whether a deadline has passed;
  timestamp_deadline_us() computes a future deadline given an offset.

DuplicateFilter.hpp / DuplicateFilter.cpp
  128-entry sliding-window dedup keyed on (source_id, message_id) pairs;
  evicts the oldest entry on overflow. Used by DeliveryEngine on the inbound
  path for RELIABLE_RETRY messages.

AckTracker.hpp / AckTracker.cpp
  Fixed-capacity (32-slot) table tracking outstanding reliable messages;
  records ACK deadlines, detects timeouts via sweep_expired(), and removes
  entries on ACK receipt via on_ack(). States: FREE / PENDING / ACKED.

RetryManager.hpp / RetryManager.cpp
  Retry scheduler with per-message exponential backoff (initial interval
  doubles each attempt, capped at 60 s); integrates with AckTracker to cancel
  retries on ACK via on_ack(); collect_due() returns messages ready for
  re-send. States: INACTIVE / WAITING / DUE / EXHAUSTED / EXPIRED.

DeliveryEngine.hpp / DeliveryEngine.cpp
  Per-channel coordinator that ties transport, ACK tracking, retry, and dedup
  together. Assigns message IDs, applies reliability semantics (BEST_EFFORT /
  RELIABLE_ACK / RELIABLE_RETRY), drives retries via pump_retries(), and
  sweeps ACK timeouts via sweep_ack_timeouts().

SOURCE — src/platform/   (OS/socket layer; adapters only, no app policy)
-------------------------------------------------------------------------
PrngEngine.hpp / PrngEngine.cpp
  Seedable xorshift64 PRNG used for deterministic impairment decisions;
  exposes next() (uint64_t), next_double() [0.0, 1.0), and next_range(lo, hi).

ImpairmentConfig.hpp
  POD configuration struct for all impairment parameters: enabled flag, fixed
  latency, jitter mean/variance, loss probability, duplication probability,
  reordering window size, partition gap/duration, and PRNG seed.
  Provides impairment_config_default() inline helper.

ImpairmentEngine.hpp / ImpairmentEngine.cpp
  Applies configurable network impairments to messages on the outbound path:
  packet loss (drop), duplication (extra copy queued), fixed latency and
  jitter (delay buffer, 32 slots), reordering (window-based release), and
  link partitions (state machine: NO_PARTITION / PARTITION_ACTIVE).
  process_inbound() handles reordering on the receive path. Fully
  reproducible given the same PRNG seed.

SocketUtils.hpp / SocketUtils.cpp
  POSIX socket primitives: socket creation, bind, connect-with-timeout,
  send_all (retry on EINTR/short write), recv_exact, sendto/recvfrom wrappers,
  and length-prefixed TCP framing helpers (write_framed / read_framed).

TcpBackend.hpp / TcpBackend.cpp
  TransportInterface over TCP; server mode supports up to 8 simultaneous
  clients via a dedicated accept thread and per-connection receive threads;
  client mode connects with timeout. Uses length-prefixed framing,
  ImpairmentEngine, and a RingBuffer inbound queue.

UdpBackend.hpp / UdpBackend.cpp
  TransportInterface over UDP datagrams; binds a single socket, serializes
  each MessageEnvelope to the wire format, and applies ImpairmentEngine before
  send. Receive path deserializes from raw datagrams.

LocalSimHarness.hpp / LocalSimHarness.cpp
  In-process simulation transport; two instances are linked via link(). Messages
  sent from one are injected directly into the peer's RingBuffer via inject()
  after passing through the ImpairmentEngine — no sockets or threads involved.
  Used for all unit tests and deterministic simulation scenarios.

SOURCE — src/app/   (application orchestration)
------------------------------------------------
Server.cpp
  Demo TCP echo server: binds on a configured port, receives DATA messages,
  logs them, sends echo replies, and pumps retries/ACK-timeouts each
  iteration. Runs until terminated.

Client.cpp
  Demo TCP client: connects to the server, sends five RELIABLE_RETRY messages
  with "Hello from client #N" payloads, waits for echo replies (up to 20
  bounded receive attempts per message), and reports the final send/echo count.
